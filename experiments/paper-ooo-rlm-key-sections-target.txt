--- SECTION abstract paper/main.tex:42-75 ---
42: \begin{abstract}
43: The immediate value of Recursive Language Models (RLMs) extends beyond
44: training a new recursive checkpoint. We argue that the RLM pattern can be
45: \emph{runtime-lifted} into an execution contract inside an existing agent
46: runtime, allowing ordinary provider-swappable LLM calls to inherit the
47: operational controls associated with recursion. RLM-FORGE implements this
48: claim by mapping RLM sub-calls onto the existing RPC mechanism of Hermes
49: Agent. Ouroboros owns recursive state, scheduling, trace replay, and evidence
50: validation. The result is a deployable runtime primitive for recursive
51: inference, not a new model architecture. The primitive preserves the
52: operational parts of RLM that matter most, including bounded context
53: partitioning, explicit sub-call graphs, replayable intermediate evidence,
54: and parent synthesis that can be rejected before state mutation. We introduce
55: TraceGuard, a deterministic evidence gate for parent synthesis, and evaluate
56: it with a claim-aware omitted-fact safety scorer. The live Hermes truncation
57: fixture shows score parity between vanilla and recursive execution after
58: scorer correction. A 24-cell live primary portability run across Hermes+GLM,
59: Claude Code, and Codex completes with 24 mandatory contract passes. The
60: same primary matrix also passes with experimental read-write operational
61: memory enabled. Memory is admitted only as a schema and retry prior, while
62: TraceGuard evidence is rebuilt from fresh child outputs in the current run.
63: We also report an observed layered-memory effect: because the Hermes adapter
64: does not pass \texttt{--ignore-rules}, Hermes' built-in \texttt{MEMORY.md}
65: can be injected before the explicit RLM-FORGE envelope. Deterministic
66: memory-runtime benchmarks then isolate a narrow performance benefit:
67: guarded memory rejects answer-memory contamination, separates Hermes-style
68: prompt memory from RLM-FORGE guarded memory, and reduces repeated repair calls
69: under generated schema-failure profiles. The strongest supported statement is
70: deliberately bounded: memory can improve runtime control and repair efficiency
71: without becoming admissible factual evidence. The systems claim is not that
72: RLM-FORGE wins a benchmark. It is that RLM-FORGE turns RLM from a research-time
73: inference idea into a runtime surface that existing agent infrastructure can
74: execute, inspect, and enforce.
75: \end{abstract}

277: \subsection{Two layers, one orchestration owner}
278: \label{sec:layers}
279: 
280: The integration has two layers. Ouroboros is the recursion controller and
281: owns scheduling, termination, guardrails, and trace persistence. Hermes is
282: the inner LM, called through the existing adapter for one bounded
283: inference at a time. Hermes never recursively calls Ouroboros or invokes
284: \texttt{ooo} commands as part of the RLM loop. This constraint is encoded
285: both in the system prompt sent to Hermes and in the structured contract that
286: validates its responses.
287: 
288: This ownership split is what lets RLM-FORGE inherit the control advantage of
289: RLM without pretending Hermes itself has become a new recursive model. Hermes
290: does local work. Ouroboros owns the recursive state machine. TraceGuard decides
291: whether a parent is allowed to claim what its children observed.
292: 
293: Each outer-to-inner handoff is a single envelope rendered as the prompt
294: passed through the \texttt{HermesCliRuntime} adapter. The envelope carries
295: the following fields.
296: 
297: \begin{itemize}[leftmargin=*]
298: \item \texttt{mode}. One of \texttt{decompose\_ac}, \texttt{execute\_atomic},
299:       \texttt{summarize\_chunk}, or \texttt{synthesize\_parent}.
300: \item \texttt{call\_context}. The values \texttt{call\_id},
301:       \texttt{parent\_call\_id}, \texttt{rlm\_node\_id}, and
302:       \texttt{ac\_node\_id}.
303: \item \texttt{constraints}. The flag \texttt{must\_not\_call\_ouroboros} set
304:       to \texttt{true}, the ambiguity threshold, and the depth cap.
305: \item \texttt{context}. Only the chunks, summaries, ancestry, and child
306:       results selected by the outer scaffold.
307: \item \texttt{evidence\_handles}. The set of identifiers Hermes is
308:       permitted to cite.
309: \end{itemize}
310: 
311: The response carries \texttt{verdict}, \texttt{confidence}, \texttt{result},
312: evidence references, and residual gaps. All of this is validated before the
313: outer scaffold mutates state.

--- SECTION operational_memory_and_benchmarks paper/main.tex:477-591 ---
477: \subsection{Operational memory experiment}
478: \label{sec:memory-experiment}
479: 
480: RLM-FORGE also evaluates an experimental memory-shaped recursion path. The
481: memory backend is a local JSONL store with an allowlisted schema. It accepts
482: only operational records such as provider schema stability, missing-handle
483: repair hints, task family, provider family, outcome, and confidence. It
484: rejects fixture answers, fact IDs, chunk IDs, raw fixture paths, prompt
485: injection text, unknown fields, and other evidence-bearing content.
486: 
487: Memory recall can enter child extraction and parent synthesis prompts only as
488: structured \texttt{memory\_priors}. These priors can recommend policies such
489: as preserving \texttt{fact\_id} with \texttt{evidence\_chunk\_id}. They are
490: not part of the accepted evidence manifest. For every cell, the manifest
491: given to TraceGuard is rebuilt from the current run's child outputs and then
492: intersected with the fixture contract. Provider-normalized child text is
493: accepted only when it matches the canonical fixture text or the same text
494: with the \texttt{FACT:<id>} prefix removed. This preserves a fresh-evidence
495: boundary while tolerating harmless provider formatting differences.
